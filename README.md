## 📦 Установка

### 1. Клонирование репозитория

```bash
cd legitVPN
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Linux/Mac
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Конфигурация

Создайте файл `.env` в корне проекта с следующим содержимым:

```env
# Telegram Bot Settings
BOT_TOKEN=xxxxxxx:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ADMIN_IDS=123456789,987654321

# Marzneshin API Settings
MARZNESHIN_API_URL=http://your_marzneshin_panel:8000
MARZNESHIN_ADMIN_USERNAME=admin_username
MARZNESHIN_ADMIN_PASSWORD=admin_password

# Service Configuration
SERVICE_ID=1
INBOUND_ID=1

# Database Settings
DATABASE_URL=sqlite:///./bot.db

# Pricing (in rubles)
PRICE_1_MONTH=99
PRICE_3_MONTHS=279
PRICE_6_MONTHS=579
PRICE_12_MONTHS=999

# Trial Settings
TRIAL_DAYS=3

# Referral System
REFERRAL_BONUS_POINTS=10
POINTS_TO_RUBLE_RATIO=1

# Support
SUPPORT_CHAT_URL=t.me/vpnsakura
```

## 🚀 Запуск


Бот начнет работать и будет готов принимать команды.

## 👥 Функционал пользователей (обычные)

### Основные команды:

#### `/start`
- Показывает приветственное сообщение
- Кнопки: "Пробный период", "Оформить подписку"
- Поддерживает реферальные ссылки

#### `/account`
- Показывает информацию профиля:
  - Ваш Telegram ID
  - Баланс баллов
  - Дату окончания подписки
  - Количество дней до конца
- Кнопки для управления подпиской

#### `/pay`
- Выбор способа оплаты
- Выбор тарифа
- Активация подписки

#### `/update_keys`
- Обновление ключей доступа
- Удаление на всех устройствах
- Получение новой ссылки

#### `/help`
- Помощь в устранении неполадок
- Ссылка на техподдержку

### Кнопки и функции:

**Пробный период:**
- Активирует 3-дневный тестовый период
- Показывает ссылку подписки

**Оформить подписку:**
- Выбор способа оплаты (баллы, карта, СБП)
- Выбор тарифа
- Автоматическое создание подписки после оплаты

**Профиль:**
- Просмотр баланса баллов
- Статус подписки
- Управление подпиской

**Реферальная система:**
- Генерирует реферальную ссылку
- Показывает баланс баллов
- +10 баллов за каждого активного друга

## 🗄️ Модели БД

### User
- `telegram_id` - ID пользователя в Telegram
- `username` - Username в Marzneshin
- `subscription_key` - Ключ подписки
- `balance_points` - Баланс реферальных баллов
- `is_active` - Активен ли пользователь
- `created_at`, `updated_at` - Даты

### Subscription
- `telegram_id` - ID пользователя
- `username` - Username в Marzneshin
- `subscription_key` - Ключ подписки
- `start_date`, `expire_date` - Даты подписки
- `is_active` - Активна ли подписка

### Referral
- `referrer_telegram_id` - ID приглашающего
- `referred_telegram_id` - ID приглашенного
- `bonus_points` - Начисленные баллы
- `is_active` - Активен ли реферрал

### Payment
- `telegram_id` - ID пользователя
- `amount` - Сумма платежа
- `payment_method` - Способ оплаты
- `subscription_days` - Количество дней подписки
- `status` - Статус платежа

## 🔑 Ключевые особенности реализации

### Интеграция с Marzneshin API:
- Автоматическое создание пользователей
- Управление подписками
- Получение информации о системе
- Обновление и удаление ключей

### Система оплаты:
- **Баллы**: Список прямо из баланса пользователя
- **Карта/СБП**: Заглушки для дальнейшей интеграции с платежами
- **Telegram Stars**: Пока недоступна

### Реферальная система:
- Каждый новый пользователь получает реферальную ссылку
- +10 баллов за каждого активного приглашенного пользователя
- Баллы можно использовать для оплаты подписки

### Форматирование сообщений:
- Использует Markdown V2
- Красивые эмодзи
- Информативные кнопки

## 🛠️ Решение проблем

### Бот не подключается к API
- Проверьте URL Marzneshin API в .env
- Убедитесь, что учетные данные администратора верны
- Проверьте доступность API с вашего сервера

### Ошибки при создании подписки
- Проверьте SERVICE_ID и INBOUND_ID в .env
- Убедитесь, что сервис и inbound существуют на панели
- Проверьте логи для деталей ошибки

### Пользователи не получают баллы
- Проверьте REFERRAL_BONUS_POINTS в .env
- Убедитесь, что реферральный пользователь активирован
- Проверьте БД на наличие записей о рефералах
